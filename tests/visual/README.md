# Visual regression evidence

`matrix.json` is executed once per platform and currently defines 30 required baselines. Each platform covers light/dark, LTR/RTL, 1.0/1.3/2.0 font scales, narrow/wide layouts, deterministic stream checkpoints, controls, fallbacks, fullscreen, and distinct image, code, math, Mermaid, and harness states.

Android and iOS baselines were captured from the packed Expo 56 Release-Hermes fixture. `capture.mjs` refuses baseline writes unless `REVIEWED_VISUAL_BASELINE_UPDATE=1`, `VISUAL_RUNTIME`, and the candidate commit and package SHA-256 are supplied. It writes a schema-2 capture manifest containing tool-generated semantic receipts bound to the candidate, matrix, device/runtime, command outcome, and every PNG SHA-256. It does not accept or write reviewer names or decisions. A baseline is never generated from a unit-test snapshot. Comparison failures write PNGs under `tests/visual/artifacts/`, which is ignored except in CI artifact upload.

Android capture also requires a successful accessibility hierarchy assertion. iOS capture requires `maestro`; screenshot-only output is rejected. Manual VoiceOver/TalkBack remains a separate hardware gate.

`bun run visual:verify` is device-independent. It rejects malformed or stale manifests, blank or implausibly large PNGs, duplicate cases, missing or modified referenced PNGs, and unreferenced PNGs. Release verification also requires `release-review-attestation.json`, generated by the protected `release-evidence` workflow after its environment approval gate and bound to the candidate, matrix, manifests, and PNGs. Imported reviewer strings cannot satisfy this gate. A valid subset is reported as incomplete without being mislabeled corrupt. The current 24 legacy artifacts per platform are valid integrity evidence, with six current matrix cases and fresh schema-2 capture/review evidence still required.

After the complete matrix exists, `VISUAL_CASE_ID=<case-id>` may be used for a surgical baseline replacement. Partial updates validate the existing manifest and covered matrix hash and refuse to write a manifest unless every current artifact hash and receipt remains present. Any update removes the prior workflow attestation, so the protected review gate must run again.
